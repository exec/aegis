# Phase 15: Shell — Design Spec

## Goal

Deliver an interactive shell on Aegis. The shell forks and execs external programs from the initrd, implements `cd`/`exit`/`help` as true builtins, and ships eight companion programs (`ls`, `cat`, `echo`, `pwd`, `uname`, `clear`, `true`, `false`). `make test` is unchanged. `make shell` boots an interactive session.

---

## Architecture

```
kernel boots
    └─ proc_spawn_init()
           ├─ INIT=hello  →  hello.elf  → print [INIT] → exit   (make test)
           └─ INIT=shell  →  shell.elf  → REPL loop             (make shell)

shell REPL
    ├─ builtin: cd / exit / help  (runs in shell process)
    └─ external: fork() → execve("/bin/<cmd>", argv, envp) → waitpid()
                                     ↑
                              loaded from initrd
```

**Key invariant:** `make test` is bit-for-bit identical to Phase 14. The oracle does not change. The shell is never involved in the headless test path.

## Section 1: Kernel Changes

### 1.1 Process struct additions (`kernel/proc/proc.h`)

Add to `aegis_process_t`:

```c
uint32_t pid;          /* unique process ID; 1 = init */
uint32_t ppid;         /* parent PID; 0 = no parent   */
char     cwd[256];     /* current working directory; init = "/" */
uint64_t exit_status;  /* lower 8 bits = exit code, written before zombie */
```

Add to `aegis_task_t`:

```c
uint32_t state;        /* TASK_RUNNING=0  TASK_BLOCKED=1  TASK_ZOMBIE=2 */
uint32_t waiting_for;  /* PID the task is blocked waiting for; 0 = any child */
```

### 1.2 PID allocator (`kernel/proc/proc.c`)

Add a file-scoped `static uint32_t s_next_pid = 1`. Add `proc_alloc_pid()` returning `s_next_pid++`. Single-core; no locking needed.

Update `proc_spawn` to initialise the new fields:
```c
proc->pid           = proc_alloc_pid();   /* 1 for init */
proc->ppid          = 0;
proc->cwd[0]        = '/';
proc->cwd[1]        = '\0';
proc->exit_status   = 0;
proc->task.state    = TASK_RUNNING;
proc->task.waiting_for = 0;
```

### 1.3 Syscall frame exposure

#### Problem

`sys_execve` must redirect the returning process to a new RIP and RSP. On the SYSCALL/SYSRET path the CPU does not push an iretq frame — `syscall_entry.asm` saves user state onto the kernel stack manually. Locating that saved state by offset arithmetic inside `sys_execve` is fragile: a one-word error silently corrupts the wrong field and produces a `#GP` with no useful diagnostic.

#### Solution: `syscall_frame_t` + frame pointer parameter

Define in `kernel/syscall/syscall.h`:

The struct field order must match the actual push sequence in `syscall_entry.asm`. The kernel stack grows downward; the last-pushed value is at the lowest address (the struct base). The current push sequence is:

1. `push [g_user_rsp]` (deepest — highest address in frame)
2. `push rcx` (user RIP)
3. `push r11` (user RFLAGS)
4. `push r8`
5. `push r9`
6. `push r10` (shallowest among saves — lowest address = struct base)
7. `push r9` again (arg6 slot — discarded after call, **not** part of the struct)

After discarding the arg6 slot, `frame = RSP + 8`. Memory layout from struct base (low → high):

```c
typedef struct {
    uint64_t r10;       /* offset +0:  saved user r10 (Linux arg4) */
    uint64_t r9;        /* offset +8:  saved user r9  (Linux arg6) */
    uint64_t r8;        /* offset +16: saved user r8  (Linux arg5) */
    uint64_t rflags;    /* offset +24: saved user RFLAGS (r11 at syscall entry) */
    uint64_t rip;       /* offset +32: saved user RIP  (rcx at syscall entry) */
    uint64_t user_rsp;  /* offset +40: saved user RSP (pushed first = highest addr) */
} syscall_frame_t;
```

In `syscall_entry.asm`, after all saves and before the register shuffle, compute the frame pointer with `lea rdi, [rsp+8]` (rsp+8 skips the arg6 slot to point at the saved r10). Pass it as the **first** SysV argument (rdi). Shift all existing arguments by one position:

```
rdi = &syscall_frame_t   (new)
rsi = syscall number     (was rdi)
rdx = arg1               (was rsi)
rcx = arg2               (was rdx)
r8  = arg3               (was rcx — note: already overwritten by shuffle)
r9  = arg4               (was r8)
[rsp+8] = arg5           (was [rsp+8] arg6)
[rsp+16] = arg6          (was not present — new stack slot)
```

Update `syscall_dispatch` declaration to:
```c
uint64_t syscall_dispatch(syscall_frame_t *frame, uint64_t num,
                          uint64_t arg1, uint64_t arg2, uint64_t arg3,
                          uint64_t arg4, uint64_t arg5, uint64_t arg6);
```

`sys_execve` receives `frame` from `syscall_dispatch` and sets:
```c
frame->rip     = new_entry_rip;
frame->user_rsp = new_user_rsp;
```
The normal `sysretq` path in `syscall_entry.asm` restores RIP from `rcx` (popped from the saved `rip` slot) and RSP from the saved `user_rsp` slot, so the process returns directly into the new ELF without any extra mechanism.

### 1.4 `fs_base` save/restore (`kernel/arch/x86_64/syscall_entry.asm` + `kernel/sched/sched.c`)

**Correct insertion point:** immediately after `ctx_switch(old, new)` returns in `sched_tick`, at which point `s_current` has been updated to the incoming task. Write the incoming process's `fs_base` into `IA32_FS_BASE` (MSR `0xC0000100`) only if `s_current->is_user == 1`:

```c
/* In sched_tick, after ctx_switch: */
if (s_current->is_user) {
    aegis_process_t *proc = (aegis_process_t *)s_current;
    arch_set_fs_base(proc->fs_base);
}
```

`arch_set_fs_base` (already in `arch.h`) writes the MSR. This runs inside the PIT ISR with `IF=0`, so there is no concurrency risk on the `wrmsr`.

**Why not in `proc_enter_user`:** `proc_enter_user` fires only on a task's first entry; preempted tasks resume via the IRET path in `isr_common_stub` which does not reload FS.base.

### 1.5 New syscalls

#### `sys_fork` — syscall 57

Returns child PID to parent, 0 to child.

**Steps:**

1. Allocate a new `aegis_process_t` via `kva_alloc_pages(1)`.
2. Copy all `aegis_process_t` fields from parent: `fds`, `caps`, `brk`, `mmap_base`, `fs_base`, `cwd`. Set `pid = proc_alloc_pid()`, `ppid = parent->pid`, `task.state = TASK_RUNNING`, `task.waiting_for = 0`, `exit_status = 0`.
3. Allocate a new PML4 via `vmm_create_user_pml4`.
4. **Full page copy — no stack buffer:**
   - Allocate a scratch page: `uint64_t scratch_phys = pmm_alloc_page()`.
   - Walk parent PML4 entries 0–255. For each present level-4 entry, walk its PDPT entries, then PD entries, then PT leaf entries. For each present leaf PTE with a user-accessible physical page:
     - `pmm_alloc_page()` → `dst_phys`.
     - Map `src_phys` into the window → read 4096 bytes into a scratch buffer: `vmm_window_map(src_phys)` → `memcpy(tmp_kernel_buf, window, 4096)` → `vmm_window_unmap()`. Use `scratch_phys` mapped via the same window: `vmm_window_map(scratch_phys)` → `memcpy(window, tmp_kernel_buf, 4096)` → `vmm_window_unmap()`.

     **Note on buffer size:** A local `char tmp[4096]` in `sys_fork` would overflow the 4 KB user-process kernel stack. Use the scratch physical page itself as the buffer: map source → copy 64-byte cache lines using a small intermediate buffer (≤256 bytes on-stack), or map both source and scratch simultaneously using two window slots. If only one window slot exists (current implementation), map source, copy to a ≤256-byte on-stack chunked loop, map scratch, write. Alternatively, add a second window slot to `vmm.c` for this purpose. The plan task must choose one approach.

   - `vmm_map_user_page(child_pml4, va, dst_phys, same_flags_as_parent)`.
   - After all pages are copied: `pmm_free_page(scratch_phys)`.
5. Allocate a kernel stack: `uint8_t *kstack = kva_alloc_pages(1)`.
6. **Build the child's initial kernel stack frame** so that when the scheduler runs the child via `ctx_switch`, the child returns through the normal `syscall_entry` SYSRET path with RAX = 0.

   The frame must mirror the layout that `syscall_entry.asm` produces at the point where `ctx_switch` would have been called (if the scheduler preempted the parent mid-syscall). The child's kernel stack (from low to high, descending RSP):

   The layout must exactly match the `syscall_frame_t` field order so that `fork_child_return` can restore it identically to the normal `syscall_entry` tail. Child's kernel stack (low address → high address, i.e. RSP starts at r15 and grows up):

   ```
   [r15=0][r14=0][r13=0][r12=0][rbp=0][rbx=0]   ← ctx_switch callee-saves (RSP after ctx_switch ret)
   [fork_child_return]                             ← ret address for ctx_switch
   [r10 = parent's saved r10]                     ← syscall_frame_t offset +0
   [r9  = parent's saved r9]                      ← syscall_frame_t offset +8
   [r8  = parent's saved r8]                      ← syscall_frame_t offset +16
   [rflags = parent's saved RFLAGS]               ← syscall_frame_t offset +24
   [rip    = parent's saved user RIP]             ← syscall_frame_t offset +32
   [user_rsp = parent's saved user RSP]           ← syscall_frame_t offset +40
   ```

   `fork_child_return` is a label in `syscall_entry.asm` that mirrors the tail of `syscall_entry` exactly:
   ```nasm
   fork_child_return:
       pop  r10          ; restore user r10
       pop  r9           ; restore user r9
       pop  r8           ; restore user r8
       pop  r11          ; restore RFLAGS
       pop  rcx          ; restore user RIP
       mov  rax, 0       ; child returns 0 from fork()
       pop  rsp          ; restore user RSP
       sysretq
   ```

   The parent's saved user RIP, RSP, RFLAGS, r8, r9, r10 are read from the `syscall_frame_t *frame` pointer that `sys_fork` receives as its first argument — this is the exact pointer to the parent's live kernel stack frame.

7. `proc->task.rsp = (uint64_t)sp`, `stack_base = kstack`, `kernel_stack_top`, `tid`, `is_user = 1`, `stack_pages = 1`.
8. `sched_add(&proc->task)`.
9. Return `child->pid` to parent.

**fd copy note:** `fds[]` is copied by value. The `priv` pointer in each fd points into driver-owned data (console ring, kbd ring buffer, initrd entry). All three pre-opened driver `close` ops must be verified to be no-ops or ref-safe. For Phase 15, console and kbd close are no-ops; this is acceptable. Reference counting is deferred.

#### `sys_execve` — syscall 59

Signature (Linux ABI): `execve(const char *path, char *const argv[], char *const envp[])`

**Steps:**

1. Copy `path` string from user (≤255 bytes) via `copy_from_user`.
2. Copy `argv` pointer array from user: walk until NULL pointer, max 64 entries. For each non-NULL pointer, `copy_from_user` the string into kernel buffers. Preserve argc count.
3. Look up `path` in initrd via `vfs_open`-equivalent; if not found, return `-ENOENT`.
4. **Free all user pages:** Call a new helper `vmm_free_user_pages(proc->pml4_phys)` that walks PML4 entries 0–255 → PDPT entries → PD entries → PT leaf entries, calling `pmm_free_page(phys)` and clearing each PTE. Does **not** free the page-table pages themselves nor the PML4 (those remain in place for reuse). Flush TLB after (`invlpg` on each unmapped VA, or reload CR3).
5. Reset: `proc->brk = 0`, `proc->mmap_base = 0x0000700000000000ULL`, `proc->fs_base = 0`.
6. Load new ELF via `elf_load(proc->pml4_phys, data, len, &brk_start)`. Set `proc->brk = brk_start`.
7. Allocate + map a new user stack page (`USER_STACK_PAGE`); call `vmm_zero_page(phys)`.
8. **Build ABI initial stack** at `USER_STACK_TOP`. The x86-64 SysV ABI requires RSP at `_start` to point to `argc`, with the following layout ascending from RSP:

   ```
   [rsp+0]             = argc (uint64_t)
   [rsp+8]             = argv[0] pointer
   ...
   [rsp+8*(argc+1)]    = NULL  (argv terminator)
   [rsp+8*(argc+2)]    = NULL  (envp[0] — empty environment)
   [rsp+8*(argc+3)]    = AT_NULL (0)  (auxv terminator)
   ... string data ...
   ```

   Copy argv strings into the stack page at the top (below `USER_STACK_TOP`), working downward. Then write the pointer table below that, then `argc`. RSP must be 16-byte aligned at the point of the `call _start` instruction (musl's linker script arranges this), which means RSP at `_start` must be 8-byte aligned (16n+8 because `call` will push the return address). Align accordingly.

9. Modify the `syscall_frame_t *frame` (received via the `syscall_dispatch` first parameter):
   ```c
   frame->rip      = entry_rip;
   frame->user_rsp = new_user_rsp;  /* RSP pointing to argc */
   ```
   Normal `sysretq` return redirects to the new ELF.

#### `sys_waitpid` — syscall 61

Signature: `waitpid(pid_t pid, int *wstatus, int options)`

Supported: `pid == -1` (any child), `pid > 0` (specific child). `options` 0 or `WNOHANG`.

**Steps:**

1. Scan the scheduler run queue (`aegis_task_t *` pointers). For each entry, cast to `aegis_process_t *` before accessing `pid`/`ppid`. Check: `task->state == TASK_ZOMBIE` and `proc->ppid == current_proc->pid` and (`pid == -1` or `pid == proc->pid`).
2. If zombie found: read `exit_status`; if `wstatus != NULL`, `copy_to_user(wstatus, &status, sizeof(int))`; remove from run queue; free zombie's kernel stack via `kva_free_pages`, free zombie's PML4 via `vmm_free_user_pml4`; free zombie's PCB via `kva_free_pages`; return child PID.
3. If no zombie and `WNOHANG`: return 0.
4. If no zombie and blocking: call `sched_block(current_pid_or_any, pid)` (see Section 1.6), which marks the task `TASK_BLOCKED`, sets `task->waiting_for = pid`, unlinks from run queue, and yields. When rescheduled, retry from step 1.

**Interrupt safety:** `sched_block` and `sched_wake` run with `IF=0` (SFMASK clears IF on `syscall` entry; the PIT ISR also runs with IF=0 via the interrupt gate). The run-queue modification in `sched_wake` (called from `sched_exit` in the child) is therefore serialised against PIT ticks. This must be documented in `sched.c`.

#### `sys_getcwd` — syscall 79

Copy `proc->cwd` into user buffer via `copy_to_user`. Return bytes written including null terminator.

#### `sys_chdir` — syscall 80

Copy path from user (≤255 bytes). For Phase 15: accept any path without existence validation; copy into `proc->cwd`. Return 0. (Full validation deferred to Phase 16 when a real filesystem is present.)

#### `sys_getppid` — syscall 110

Return `proc->ppid`.

#### `sys_getdents64` — syscall 217

Signature: `getdents64(int fd, void *dirp, size_t count)`

The initrd registers virtual directory entries for `/`, `/bin`, and `/etc`. `sys_open` on one of these paths returns an fd with a `readdir` op. Each call to `getdents64` fills the user buffer with `linux_dirent64` entries:

```c
struct linux_dirent64 {
    uint64_t d_ino;    /* 1-based file index */
    int64_t  d_off;    /* next entry offset (opaque; just use index+1) */
    uint16_t d_reclen; /* sizeof header + strlen(d_name) + 1, rounded to 8 */
    uint8_t  d_type;   /* DT_REG=8, DT_DIR=4 */
    char     d_name[]; /* null-terminated name */
};
```

The per-fd read position (`vfs_file_t.offset`) is **reused as the directory entry index** — the same field, different semantics. This is safe because `read` and `getdents64` are mutually exclusive operations on a given fd: you open a directory as a directory (not for byte-reading). The `read` op for directory fds must return `EISDIR`.

`sys_getdents64` checks that `fd.ops->readdir != NULL` before calling (returns `ENOTDIR` if NULL). Returns total bytes written; 0 means end of directory.

**`vfs_ops_t` extension:**

Add `int (*readdir)(void *priv, uint64_t index, char *name_out, uint8_t *type_out)` to `vfs_ops_t`. Returns 0 on success, -1 on end of directory. All existing ops instances (`initrd_ops`, `console_ops`, `kbd_ops`) must be updated to zero-initialise this field. Under C99 designated initializers with zero-init of unlisted fields, omitting `readdir` implicitly sets it to NULL — but every `vfs_ops_t` initializer must be audited and an explicit `NULL` added to make this intentional, not accidental.

### 1.6 Scheduler changes

Add to `kernel/sched/sched.c`:

- `TASK_RUNNING = 0`, `TASK_BLOCKED = 1`, `TASK_ZOMBIE = 2` constants (or enum).
- `void sched_block(uint32_t waiting_for_pid)`: sets `s_current->state = TASK_BLOCKED`, `s_current->waiting_for = waiting_for_pid`, unlinks current from the circular run queue, calls the next task via the idle/yield path.
- `void sched_wake(aegis_task_t *task)`: sets `task->state = TASK_RUNNING`, re-links into the run queue before `s_current`.

**`sched_tick` zombie/blocked skip:**

```c
/* Advance s_current, skipping non-runnable tasks.
 * The loop terminates because task_idle is always TASK_RUNNING. */
aegis_task_t *old = s_current;
do {
    s_current = s_current->next;
} while (s_current->state != TASK_RUNNING);
ctx_switch(old, s_current);
```

`task_idle` (the permanent kernel idle task, added in Phase 10) is always `TASK_RUNNING` and never zombies, ensuring the loop terminates.

**`sched_exit` zombie + wakeup:**

```c
/* Mark self zombie */
aegis_process_t *dying = (aegis_process_t *)s_current;
s_current->state = TASK_ZOMBIE;
dying->exit_status = exit_code;

/* Wake blocked parent, if any */
aegis_task_t *t = s_current->next;
while (t != s_current) {
    aegis_process_t *p = (aegis_process_t *)t;
    if (t->is_user && t->state == TASK_BLOCKED &&
        p->pid == dying->ppid &&
        (t->waiting_for == 0 || t->waiting_for == dying->pid)) {
        sched_wake(t);
        break;
    }
    t = t->next;
}

/* Yield — zombie stays in queue until parent calls waitpid.
 * sched_yield_to_next() advances s_current to the next RUNNING task
 * and calls ctx_switch(dying_zombie, s_current). The zombie's kernel
 * stack is still live during ctx_switch (it writes dying->task.rsp).
 * After ctx_switch returns in the new task, the zombie's stack is no
 * longer live and may be freed by waitpid at any future time.
 * This is the same deferred-cleanup guarantee as the existing sched_exit. */
sched_yield_to_next();   /* defined in sched.c: advances s_current past non-RUNNING tasks, ctx_switch */
```

## Section 2: Initrd

### 2.1 Binary file support

`initrd_entry_t.data` is `const char *`. ELF binaries are embedded via `xxd -i prog.elf`, which produces `unsigned char prog_elf[]` and `unsigned int prog_elf_len`. Cast `(const char *)prog_elf` for the `data` field and `prog_elf_len` for size. No null terminator is added or expected.

### 2.2 Directory entries

The initrd driver registers three virtual directory fds:

| Path | Contents |
|------|----------|
| `/` | `etc`, `bin` |
| `/etc` | `motd` |
| `/bin` | `sh`, `ls`, `cat`, `echo`, `pwd`, `uname`, `clear`, `true`, `false` |

The `readdir` op iterates `s_files[]` and filters by prefix match.

### 2.3 File layout

```
/etc/motd
/bin/sh
/bin/ls
/bin/cat
/bin/echo
/bin/pwd
/bin/uname
/bin/clear
/bin/true
/bin/false
```

### 2.4 Kernel image size check

Each musl-linked static binary is approximately 50–150 KB. Ten programs × 100 KB average = ~1 MB of `.data`. The kernel itself is ~200 KB. Total image should be well under 4 MB. **Before implementing, verify** with `wc -c` on each compiled ELF and adjust if any binary is unexpectedly large. Strip debug info with `-s` in each program's Makefile if needed.

## Section 3: Shell (`user/shell/`)

### 3.1 REPL

```c
printf("[SHELL] Aegis shell ready\n");
while (1) {
    write(1, "# ", 2);
    if (!fgets(line, sizeof(line), stdin)) break;  /* EOF → exit */
    /* tokenize, dispatch */
}
```

On EOF (`fgets` returns NULL): `exit(0)`.

### 3.2 Builtins

| Command | Behaviour |
|---------|-----------|
| `cd [path]` | `chdir(path ?: "/")`. Print error on failure. |
| `exit [n]` | `exit(atoi(n) ?: 0)` |
| `help` | Print builtin list + "other commands in /bin/" |

### 3.3 Fork + exec

```c
char *envp[] = { NULL };
pid_t pid = fork();
if (pid == 0) {
    char full[64];
    if (argv[0][0] != '/') {
        snprintf(full, sizeof(full), "/bin/%s", argv[0]);
        execve(full, argv, envp);
    } else {
        execve(argv[0], argv, envp);
    }
    fprintf(stderr, "%s: not found\n", argv[0]);
    _exit(127);
} else if (pid > 0) {
    int status;
    waitpid(pid, &status, 0);
} else {
    perror("fork");
}
```

## Section 5: Build

### 5.1 INIT variable and symbol naming

To avoid changing `proc.c` when switching init binary, both `hello_bin.c` and `shell_bin.c` export the same symbol names: `init_elf[]` and `init_elf_len`. This is achieved by passing a custom symbol prefix to `xxd` via a wrapper, or by post-processing the generated C file with `sed` to rename the symbols. `proc_spawn_init` always references `extern const unsigned char init_elf[]` and `extern const unsigned int init_elf_len`.

### 5.2 Makefile structure

```makefile
INIT ?= hello

ifeq ($(INIT),shell)
INIT_ELF_SRC = user/shell/shell.elf
else
INIT_ELF_SRC = user/hello/hello.elf
endif

INIT_BIN_C = kernel/init_bin.c

$(INIT_BIN_C): $(INIT_ELF_SRC)
	cd $(dir $<) && xxd -i $(notdir $<) | \
	  sed 's/$(notdir $(basename $<))_elf/init_elf/g' > ../../$(INIT_BIN_C)

# Kernel ELF depends on INIT_BIN_C; INIT_BIN_C depends on the selected ELF.
# Changing INIT forces a rebuild of INIT_BIN_C and the kernel ELF.
# A stamp file records the last INIT value to detect changes:
INIT_STAMP = .init_stamp_$(INIT)
$(INIT_BIN_C): $(INIT_STAMP)
$(INIT_STAMP):
	@rm -f .init_stamp_*
	@touch $@
```

`make test` always uses `INIT=hello` (the default). `make shell` sets `INIT=shell`. The stamp file ensures that `make test` after `make shell` regenerates `init_bin.c` with the correct binary.

### 5.3 New `make shell` target

```makefile
.PHONY: shell
shell:
	$(MAKE) INIT=shell run
```

### 5.4 Program Makefiles

Each `user/<prog>/Makefile`:
```makefile
CC     = musl-gcc
CFLAGS = -static -O2 -s -fno-pie -no-pie -Wl,--build-id=none
<prog>.elf: main.c
	$(CC) $(CFLAGS) -o $@ $<
clean:
	rm -f <prog>.elf
```

## Constraints and Forward-Looking Notes

**No I/O redirection or pipes.** `>`, `<`, `|` are Phase 16.

**No `$?` or variable expansion.** Shell does not parse `$VAR`. Phase 16.

**No signal handling.** `Ctrl-C` does not kill the foreground process. Phase 16.

**`sys_chdir` does not validate path existence.** Deferred to Phase 16.

**`sys_munmap` still has no VA reclaim path.** Each fork+exec child consumes mmap VA before exec resets it. At Phase 15 scale this is negligible.

**`sys_waitpid` only supports `pid == -1` and `pid > 0`.** Process groups not implemented.

**PID counter never resets.** With a 32-bit counter and one fork per second, overflow takes ~136 years.

**`sys_fork` fd copy without reference counting.** All three pre-opened fd drivers have no-op or safe close for Phase 15. Reference counting deferred.

**ELF NX bit not set on data segments.** Pre-existing from Phase 14. Phase 16 concern.
