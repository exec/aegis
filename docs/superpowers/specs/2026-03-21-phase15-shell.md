# Phase 15: Shell — Design Spec

## Goal

Deliver an interactive shell on Aegis. The shell forks and execs external programs from the initrd, implements `cd`/`exit`/`help` as true builtins, and ships eight companion programs (`ls`, `cat`, `echo`, `pwd`, `uname`, `clear`, `true`, `false`). `make test` is unchanged. `make shell` boots an interactive session.

---

## Architecture

### Overview

```
kernel boots
    └─ proc_spawn_init()
           ├─ INIT=hello  →  hello.elf  → print [INIT] → exit   (make test)
           └─ INIT=shell  →  shell.elf  → REPL loop             (make shell)

shell REPL
    ├─ builtin: cd / exit / help  (runs in shell process)
    └─ external: fork() → execve("/bin/<cmd>", argv, envp) → waitpid()
                                     ↑
                              loaded from initrd
```

### Key invariant

`make test` remains bit-for-bit identical. The oracle does not change. The shell is never involved in the headless test path.

## Section 1: Kernel Changes

### 1.1 Process struct (`kernel/proc/proc.h`)

Add to `aegis_process_t`:

```c
uint32_t pid;          /* unique process ID; 1 = init */
uint32_t ppid;         /* parent PID; 0 = no parent   */
char     cwd[256];     /* current working directory; init = "/" */
uint64_t exit_status;  /* set by sched_exit before zombie       */
```

### 1.2 Task state (`kernel/sched/sched.h`)

Add to `aegis_task_t`:

```c
uint32_t state;        /* TASK_RUNNING=0  TASK_BLOCKED=1  TASK_ZOMBIE=2 */
```

Add a global PID counter (starts at 1, incremented atomically on each fork).

### 1.3 fs_base save/restore (`kernel/arch/x86_64/syscall_entry.asm`)

On every `ctx_switch` to a user task, write `proc->fs_base` into `IA32_FS_BASE` (MSR `0xC0000100`) via `wrmsr`. This fixes the Phase 14 single-process TLS assumption before a second user process can run.

Pattern (inserted at the start of the return-to-user path in `proc_enter_user` or equivalently in `sched_tick` just before `ctx_switch`):

```nasm
; load proc pointer (kva VA of aegis_process_t for the incoming task)
; read proc->fs_base into edx:eax
; mov ecx, 0xC0000100
; wrmsr
```

### 1.4 New syscalls

#### `sys_fork` — syscall 57

Returns child PID to parent, 0 to child.

Steps:
1. Allocate a new `aegis_process_t` via `kva_alloc_pages`.
2. Copy all fields from parent (`fds`, `caps`, `brk`, `mmap_base`, `fs_base`, `cwd`).
3. Assign a new PID from the global counter; set `ppid = parent->pid`.
4. Allocate a new PML4 via `vmm_create_user_pml4`.
5. Walk parent PML4 entries 0–255. For each present leaf PTE: `pmm_alloc_page`, `vmm_zero_page` not needed (we `memcpy` contents via window), `vmm_map_user_page` into child PML4 with same flags.
6. Allocate a kernel stack via `kva_alloc_pages`; build the initial kernel stack frame identically to `proc_spawn` (iretq frame + `proc_enter_user` return address + callee-saved zeros). The child's `task.rsp` points at this frame — it will return from the `ctx_switch` call site in the scheduler and appear to have just returned from `syscall_dispatch` with return value 0.
7. Add child task to scheduler run queue via `sched_add`.
8. Return `child->pid` to parent.

**Page copy via window:** `vmm_window_map(src_phys)` → `memcpy(tmp, window, 4096)` → `vmm_window_unmap()` → `vmm_window_map(dst_phys)` → `memcpy(window, tmp, 4096)` → `vmm_window_unmap()`. Use a 4KB kernel stack buffer as `tmp`.

#### `sys_execve` — syscall 59

Signature (Linux ABI): `execve(const char *path, char *const argv[], char *const envp[])`

Steps:
1. Copy `path` from user (≤255 bytes) via `copy_from_user`.
2. Copy `argv` pointer array from user (walk until NULL, max 64 entries); copy each string.
3. Look up `path` in initrd via `vfs_open`; if not found, return `-ENOENT`.
4. Free all user pages: walk PML4 entries 0–255, for each present leaf PTE call `pmm_free_page` and `vmm_unmap_user_page`.
5. Reset `proc->brk` to 0, `proc->mmap_base` to `0x0000700000000000`, `proc->fs_base` to 0.
6. Load ELF segments from initrd data via `elf_load` (existing function).
7. Allocate + map a new user stack page (`USER_STACK_PAGE`); zero it via `vmm_zero_page`.
8. Build ABI initial stack at `USER_STACK_TOP - 128`:
   - String block: copy argv strings contiguously just below `USER_STACK_TOP`.
   - Pointer array: push argv pointers, then NULL, then NULL (empty envp), then NULL.
   - `argc` = number of argv entries.
   - Final RSP points to `argc`.
9. Modify the saved iretq frame on the current kernel stack (RIP ← entry point, RSP ← new user RSP). Return from syscall — the iretq resumes execution at the new ELF entry.

No new process is created; the calling process's address space is replaced in place.

#### `sys_waitpid` — syscall 61

Signature: `waitpid(pid_t pid, int *wstatus, int options)`

Steps:
1. Only `pid == -1` (wait any child) and `pid > 0` (wait specific child) are supported. `options` must be 0 or `WNOHANG`.
2. Scan run queue for a child with matching PID and `state == TASK_ZOMBIE`.
3. If zombie found: copy `exit_status` to user `*wstatus`; free zombie's kernel stack + PCB via `kva_free_pages`; free zombie's PML4 via `vmm_free_user_pml4`; remove zombie from run queue; return child PID.
4. If no zombie and `WNOHANG`: return 0.
5. If no zombie and blocking: set `current->state = TASK_BLOCKED`; record `current->waiting_for = pid`; call `sched_yield()` (removes current from active ring, loops until re-added). When woken by child's exit, retry from step 2.

**Child exit wakeup** (in `sched_exit`): before marking self as zombie, scan run queue for parent; if parent `state == TASK_BLOCKED` and `waiting_for` matches, set parent `state = TASK_RUNNING` and re-add to run queue.

#### `sys_chdir` — syscall 80

Copy path from user; validate length ≤ 255. For Phase 15, accept any path (no real directory existence check — initrd is flat). Copy into `proc->cwd`. Return 0.

#### `sys_getcwd` — syscall 79

Copy `proc->cwd` into user buffer (up to `size` bytes) via `copy_to_user`. Return number of bytes written.

#### `sys_getppid` — syscall 110

Return `proc->ppid`.

#### `sys_getdents64` — syscall 217

Signature: `getdents64(int fd, void *dirp, size_t count)`

The initrd gets a new virtual directory entry. `sys_open("/")` or `sys_open("/bin")` returns an fd whose ops have a `readdir` callback. Each call to `getdents64` fills the user buffer with one or more `linux_dirent64` structs:

```c
struct linux_dirent64 {
    uint64_t d_ino;
    int64_t  d_off;
    uint16_t d_reclen;
    uint8_t  d_type;   /* DT_REG=8, DT_DIR=4 */
    char     d_name[];
};
```

The initrd driver maintains a per-fd offset (index into `s_files[]`). Each call advances the offset. Returns total bytes written; 0 means end of directory.

### 1.5 PID allocator

Global `static uint32_t s_next_pid = 1`. `proc_alloc_pid()` returns `s_next_pid++`. Single-core, no locking needed. Init process gets PID 1 in `proc_spawn`.

### 1.6 Scheduler changes

Add `sched_block()`/`sched_wake(aegis_task_t *)` functions:
- `sched_block()`: sets `current->state = TASK_BLOCKED`, unlinks current task from circular run queue, calls `sched_yield()` (picks next task).
- `sched_wake(task)`: sets `task->state = TASK_RUNNING`, re-links into run queue.

Zombie tasks remain in the run queue with `state == TASK_ZOMBIE` so the parent can find and collect them, but the scheduler skips them (checks `state != TASK_RUNNING` before dispatching).

## Section 2: Initrd

### 2.1 Binary file support

The current `initrd_entry_t` stores `const char *data` — this already works for binary content as long as the data is not treated as a null-terminated string. ELF data is embedded via `xxd -i`, which produces a `const unsigned char[]` and a `const unsigned int` length. Cast `const unsigned char *` to `const char *` for the `data` field.

### 2.2 Directory entries

Add two virtual directory entries to the initrd driver:
- `/` — lists all top-level names (`etc`, `bin`)
- `/bin` — lists all `/bin/*` file names
- `/etc` — lists `/etc/motd`

The `vfs_ops_t` gets a new optional `readdir` callback (NULL for non-directory fds). `sys_open` on a path ending without a file match checks for a directory match; `sys_getdents64` calls the `readdir` op.

### 2.3 File layout

```
/etc/motd          (existing text file)
/bin/sh            (shell ELF)
/bin/ls            (ls ELF)
/bin/cat           (cat ELF)
/bin/echo          (echo ELF)
/bin/pwd           (pwd ELF)
/bin/uname         (uname ELF)
/bin/clear         (clear ELF)
/bin/true          (true ELF)
/bin/false         (false ELF)
```

## Section 3: Shell (`user/shell/`)

### 3.1 REPL

```
[SHELL] Aegis shell ready
# <user input>
```

Loop:
1. Write `# ` prompt to stdout.
2. `fgets` from stdin (line-buffered, max 512 chars).
3. On EOF or error: `exit(0)`.
4. Strip trailing newline; skip blank lines.
5. Tokenize by whitespace into `argv[]` (max 16 tokens).
6. Check builtins. If not a builtin: fork → exec → wait.

### 3.2 Builtins

| Command | Behaviour |
|---------|-----------|
| `cd [path]` | `chdir(path ?: "/")`. Print error on failure. |
| `exit [n]` | `exit(atoi(n) ?: 0)` |
| `help` | Print list of builtins + "other commands in /bin/" |

### 3.3 Fork + exec

```c
pid_t pid = fork();
if (pid == 0) {
    /* child */
    char full[64];
    if (argv[0][0] != '/') {
        snprintf(full, sizeof(full), "/bin/%s", argv[0]);
        execve(full, argv, envp);
    } else {
        execve(argv[0], argv, envp);
    }
    fprintf(stderr, "%s: not found\n", argv[0]);
    exit(127);
} else {
    int status;
    waitpid(pid, &status, 0);
}
```

### 3.4 Print banner on startup

```
[SHELL] Aegis shell ready
```

This line is NOT in the `make test` oracle (shell is never spawned in test mode). It's purely for interactive sessions.

## Section 4: External Programs

All compiled with `musl-gcc -static -O2 -fno-pie -no-pie -Wl,--build-id=none`.

### `ls [path]`
```c
DIR *d = opendir(argc > 1 ? argv[1] : ".");
struct dirent *e;
while ((e = readdir(d))) printf("%s\n", e->d_name);
closedir(d);
```

### `cat <file> [file...]`
```c
// for each file: open, read loop, write to stdout, close
```

### `echo [args...]`
```c
for (int i = 1; i < argc; i++) {
    if (i > 1) write(1, " ", 1);
    write(1, argv[i], strlen(argv[i]));
}
write(1, "\n", 1);
```

### `pwd`
```c
char buf[256];
getcwd(buf, sizeof(buf));
puts(buf);
```

### `uname`
```c
puts("Aegis x86_64");
```

### `clear`
```c
write(1, "\033[2J\033[H", 7);
```

### `true`
```c
return 0;
```

### `false`
```c
return 1;
```

## Section 5: Build

### Makefile additions

```makefile
INIT ?= hello

ifeq ($(INIT),shell)
INIT_BIN_C = kernel/shell_bin.c
INIT_ELF   = user/shell/shell.elf
else
INIT_BIN_C = kernel/hello_bin.c
INIT_ELF   = user/hello/hello.elf
endif

# New targets
.PHONY: shell
shell: INIT=shell
shell: run

# Build each program
PROG_ELFS = user/ls/ls.elf user/cat/cat.elf user/echo/echo.elf \
            user/pwd/pwd.elf user/uname/uname.elf user/clear/clear.elf \
            user/true/true.elf user/false/false.elf user/shell/shell.elf

$(PROG_ELFS):
	$(MAKE) -C $(dir $@)
```

Each program directory has its own minimal Makefile using `musl-gcc -static -O2 -fno-pie -no-pie -Wl,--build-id=none`.

The initrd source (`kernel/fs/initrd.c`) is generated or updated to include all `xxd`-embedded ELFs.

## Section 6: Testing

### `make test` (unchanged)

- Oracle: identical to Phase 14. `[INIT] Hello from musl libc!` + `[AEGIS] System halted.`
- `INIT` defaults to `hello`. Shell is never spawned.

### Interactive testing (`make shell`)

- Sets `INIT=shell`, compiles `kernel/shell_bin.c` as init.
- Boots QEMU with VGA + keyboard (`make run` flags).
- User sees `[SHELL] Aegis shell ready` prompt and a `#` cursor.
- Manual test checklist:
  - `ls /bin` → lists all nine programs
  - `cat /etc/motd` → prints motd content
  - `echo hello world` → prints `hello world`
  - `pwd` → prints `/`
  - `cd /bin && pwd` → prints `/bin`
  - `uname` → prints `Aegis x86_64`
  - `clear` → clears VGA screen
  - `true ; echo $?` → prints `0` (once `$?` is available, Phase 16)
  - `exit` → returns to kernel halt

## Constraints and Forward-Looking Notes

**No I/O redirection or pipes.** `>`, `<`, `|` are Phase 16. The shell does not parse them.

**No `$?` or environment variables.** The shell tokenizes but does not expand `$VAR`. Phase 16.

**No signal handling.** `Ctrl-C` does not kill the foreground process. Phase 16.

**`mmap_base` bump allocator still has no free path.** Each fork + exec cycle consumes mmap VA in the child before exec frees it. At Phase 15 scale this is negligible.

**`sys_execve` modifies the kernel stack in-place.** The iretq frame is overwritten to point at the new ELF entry and new user RSP. This is only safe because execve does not return through normal C call frames — it jumps straight to iretq.

**`sys_chdir` does not validate path existence.** A `chdir("/nonexistent")` will succeed. Full validation requires directory existence checks in the initrd driver; defer to Phase 16 when a real filesystem is considered.

**`waitpid` only supports `pid == -1` and `pid > 0`.** Process groups (`pid == 0`, `pid < -1`) are not implemented.

**PID counter never resets.** With a 32-bit counter and one fork per second, overflow takes ~136 years.
