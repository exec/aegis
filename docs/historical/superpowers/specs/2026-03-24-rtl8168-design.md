# RTL8168 Network Driver — Design Spec (Phase 27A)

## Goal

Add a bare-metal Ethernet driver for the Realtek RTL8111/8168 (PCI `10ec:8168`) to replace virtio-net on real hardware. Register up to two NICs as `eth0`/`eth1` through the existing `netdev_t` abstraction. Enable the Phase 25 network stack (ARP/IP/ICMP/TCP/UDP) to function on real hardware.

## Context

- Target hardware: machine with dual RTL8168 (PCI `02:00.0`, `03:00.0`) and Samsung NVMe (`144d:a80c`)
- Phase 27B will add RTL8125 (PCI `10ec:8125`) for the dev machine; shared code will be factored then
- `make test` uses QEMU `-machine pc` — no RTL8168 present; driver must be fully silent if device not found
- Reference: `reference/rtl8168/r8169_main.c` (Linux 6.x kernel driver)

---

## rtl_common.h

Shared between RTL8168 now and RTL8125 in Phase 27B.

### Register offsets (MMIO base + offset)

```c
#define RTL_MAC0                0x00  /* MAC bytes 0-3 (32-bit read) */
#define RTL_MAC4                0x04  /* MAC bytes 4-5 (32-bit read) */
#define RTL_MAR0                0x08  /* Multicast filter low  (32-bit) */
#define RTL_MAR4                0x0C  /* Multicast filter high (32-bit) */
#define RTL_TXDESC_ADDR_LOW     0x20  /* TX descriptor ring base (low 32) */
#define RTL_TXDESC_ADDR_HIGH    0x24  /* TX descriptor ring base (high 32) */
#define RTL_CHIPCMD             0x37  /* Chip command (8-bit) */
#define RTL_TXPOLL              0x38  /* TX poll doorbell (8-bit) */
#define RTL_INTRMASK            0x3C  /* Interrupt mask   (16-bit, RTL8168) */
#define RTL_INTRSTATUS          0x3E  /* Interrupt status (16-bit, RTL8168) */
#define RTL_TXCONFIG            0x40  /* TX configuration (32-bit) */
#define RTL_RXCONFIG            0x44  /* RX configuration (32-bit) */
#define RTL_CFG9346             0x50  /* Config register lock (8-bit) */
#define RTL_RXMAXSIZE           0xDA  /* Max RX frame size (16-bit) */
#define RTL_CPLUSSCMD           0xE0  /* C+ command (16-bit) */
#define RTL_RXDESC_ADDR_LOW     0xE4  /* RX descriptor ring base (low 32) */
#define RTL_RXDESC_ADDR_HIGH    0xE8  /* RX descriptor ring base (high 32) */
#define RTL_MAXTXPKTSIZE        0xEC  /* TX early-TX threshold (8-bit) */
```

### ChipCmd bits
```c
#define RTL_CMD_RESET   0x10u
#define RTL_CMD_RXEN    0x08u
#define RTL_CMD_TXEN    0x04u
```

### Cfg9346 unlock/lock values
```c
#define RTL_CFG9346_UNLOCK  0xC0u  /* unlock config registers for writes */
#define RTL_CFG9346_LOCK    0x00u  /* re-lock config registers */
```

### TxPoll value
```c
#define RTL_NPQ  0x40u  /* Normal Priority Queue — kick TX ring scan */
```

### TxConfig value
```c
/* TX_DMA_BURST=7 (unlimited), InterFrameGap=3 (shortest) */
#define RTL_TXCONFIG_VAL  ((7u << 8) | (3u << 24))
```

### RxConfig bits
```c
#define RTL_RX128_INT_EN   (1u << 15)
#define RTL_RX_MULTI_EN    (1u << 14)
#define RTL_RX_EARLY_OFF   (1u << 11)
#define RTL_RX_DMA_BURST   (7u << 8)
#define RTL_ACCEPT_BCAST   0x08u
#define RTL_ACCEPT_MYPHYS  0x02u
```

### CPlusCmd bits
```c
#define RTL_CPLS_RXCHKSUM  (1u << 5)
```

### Descriptor format (TX and RX identical)
```c
typedef struct __attribute__((packed)) {
    uint32_t opts1;  /* bit31=DescOwn, bit30=RingEnd, bit29=FirstFrag,
                        bit28=LastFrag, bits[0:13]=frame length */
    uint32_t opts2;  /* 0 for basic operation (no VLAN/checksum offload) */
    uint64_t addr;   /* physical DMA address of frame buffer */
} rtl_desc_t;

#define RTL_DESC_OWN        (1u << 31)  /* NIC owns descriptor */
#define RTL_DESC_RING_END   (1u << 30)  /* last descriptor in ring */
#define RTL_DESC_FIRST_FRAG (1u << 29)  /* first fragment of packet */
#define RTL_DESC_LAST_FRAG  (1u << 28)  /* last fragment of packet */
#define RTL_DESC_LEN_MASK   0x3FFFu     /* bits[13:0] = frame length */
```

256 descriptors per ring. Ring buffer must be 256-byte aligned (one 4096-byte page satisfies this).

### Buffer size

```c
/* RX buffer size per slot. Must be >= RxMaxSize written to RTL_RXMAXSIZE.
 * 1536 = 1500 (MTU) + 14 (Ethernet header) + headroom. One PMM page per slot. */
#define RTL_RX_BUF_SIZE  1536u
```

### map_bar_mmio() inline

Replaces the static `map_bar_region()` currently in `virtio_net.c`. `virtio_net.c` will `#include "rtl_common.h"` and call `map_bar_mmio()` instead.

```c
/* Map n_pages of a BAR physical region into KVA with no-cache flags.
 * flags 0x1B = Present|Write|PWT|PCD (MMIO, no-cache). */
static inline uintptr_t
map_bar_mmio(uint64_t pa, uint32_t n_pages)
{
    uintptr_t va = (uintptr_t)kva_alloc_pages(n_pages);
    for (uint32_t i = 0; i < n_pages; i++) {
        uintptr_t pva = va + (uint64_t)i * 4096;
        vmm_unmap_page(pva);
        vmm_map_page(pva, pa + (uint64_t)i * 4096, 0x1Bu);
    }
    return va;
}
```

## rtl8168.c

### DMA page allocation helper

Identical to `virtio_net.c`'s `alloc_dma_page()`:

```c
static void
alloc_dma_page(uint64_t *phys_out, uintptr_t *virt_out)
{
    uintptr_t va = (uintptr_t)kva_alloc_pages(1);
    uint64_t  pa = kva_page_phys((void *)va);
    _memset((void *)va, 0, 4096);
    *phys_out = pa;
    *virt_out = va;
}
```

### MMIO accessors (reads from/writes to already-mapped BAR)

```c
static inline void     mmio_w8 (uint8_t *base, uint32_t off, uint8_t  v) { *(volatile uint8_t  *)(base+off) = v; }
static inline void     mmio_w16(uint8_t *base, uint32_t off, uint16_t v) { *(volatile uint16_t *)(base+off) = v; }
static inline void     mmio_w32(uint8_t *base, uint32_t off, uint32_t v) { *(volatile uint32_t *)(base+off) = v; }
static inline uint8_t  mmio_r8 (uint8_t *base, uint32_t off)             { return *(volatile uint8_t  *)(base+off); }
static inline uint32_t mmio_r32(uint8_t *base, uint32_t off)             { return *(volatile uint32_t *)(base+off); }
```

### Private state

```c
#define RTL8168_MAX_DEVS  2
#define RTL8168_RING_SIZE 256

typedef struct {
    uint8_t        *mmio;                           /* MMIO base (virtual) */
    rtl_desc_t     *tx_ring;                        /* TX descriptor ring (256 entries) */
    rtl_desc_t     *rx_ring;                        /* RX descriptor ring (256 entries) */
    uint8_t        *tx_buf[RTL8168_RING_SIZE];      /* TX bounce buffers (1 page each, virtual) */
    uint64_t        tx_buf_phys[RTL8168_RING_SIZE]; /* TX bounce buffer physical addresses */
    uint8_t        *rx_buf[RTL8168_RING_SIZE];      /* RX receive buffers (1 page each, virtual) */
    uint64_t        tx_ring_phys;
    uint64_t        rx_ring_phys;
    uint16_t        tx_head;    /* next TX slot to use */
    uint16_t        rx_head;    /* next RX slot to check */
} rtl8168_priv_t;

static rtl8168_priv_t  s_devs[RTL8168_MAX_DEVS];
static netdev_t        s_netdevs[RTL8168_MAX_DEVS];
static int             s_count = 0;
```

### Initialization sequence (per device)

**Step 1 — PCI Bus Master Enable**
Read PCI config offset 0x04 via `pcie_read32(bus,dev,fn, 0x04)`. Set bits 1 (Memory Space) and 2 (Bus Master). Write back via `pcie_write32()`. Required on real hardware — UEFI does not pre-configure Bus Master unlike QEMU/SeaBIOS.

**Step 2 — Map MMIO**
Use `pcie_device_t.bar[2]` for the MMIO base address. RTL8168 BAR layout: BAR0 is I/O space (decoded as `bar[0]=0`), BAR1 is an unused 32-bit slot (decoded as `bar[1]=0`), BAR2+BAR3 form the 64-bit MMIO region (decoded into `bar[2]`). Map 1 page via `map_bar_mmio(bar[2], 1)`. Store the returned VA as `priv->mmio`.

**Step 3 — Reset**
Write `RTL_CMD_RESET` to `RTL_CHIPCMD`. Poll until the bit clears (max 1000 iterations, bare busy-poll; add a brief `io_wait()` between iterations on future real-hardware tuning if needed — reset takes 1–5 ms on real silicon).

**Step 4 — Cfg9346 unlock**
Write `RTL_CFG9346_UNLOCK` (0xC0) to `RTL_CFG9346`. This unlocks write-protection on configuration registers. Without this, writes to descriptor ring base registers, RxMaxSize, and ChipCmd may silently fail on some RTL8168 silicon variants.

**Step 5 — Read MAC**
Read 6 bytes from MMIO offsets 0x00–0x05 into `netdev_t.mac`.

**Step 6 — Disable interrupts**
Write `0x0000` to `RTL_INTRMASK`. We use polling (100 Hz PIT), not interrupts.

**Step 7 — Init multicast filter**
Write `0xFFFFFFFF` to both `RTL_MAR0` and `RTL_MAR4`. This accepts all multicast frames. Required because the MAR registers are zero after reset, which would block broadcast reception on some variants.

**Step 8 — CPlusCmd**
Write `RTL_CPLS_RXCHKSUM` (0x20) to `RTL_CPLUSSCMD`. Must be written before ChipCmd is enabled.

**Step 9 — RxMaxSize**
Write `RTL_RX_BUF_SIZE` (1536) to `RTL_RXMAXSIZE`. Consistent with the 1536-byte receive buffer slots allocated in Step 11.

**Step 10 — Allocate TX ring and buffers**
Call `alloc_dma_page(&priv->tx_ring_phys, &tx_ring_va)`. Cast VA to `rtl_desc_t *` for `priv->tx_ring`. Set `RTL_DESC_RING_END` on slot 255's `opts1`. For each of 256 slots, call `alloc_dma_page(&buf_pa, &buf_va)`, store VA into `priv->tx_buf[i]`, and store PA into `priv->tx_buf_phys[i]`. Physical addresses are cached in `tx_buf_phys` at init so the TX hot path avoids a page-table walk per frame.

**Step 11 — Allocate RX ring and buffers**
Call `alloc_dma_page(&priv->rx_ring_phys, &rx_ring_va)`. Cast VA to `rtl_desc_t *` for `priv->rx_ring`. For each of 256 slots, call `alloc_dma_page(&buf_pa, &buf_va)`. Store VA into `priv->rx_buf[i]`. Pre-arm each slot:
```c
rx_ring[i].opts1 = RTL_DESC_OWN | RTL_RX_BUF_SIZE
                   | (i == 255 ? RTL_DESC_RING_END : 0);
rx_ring[i].opts2 = 0;
rx_ring[i].addr  = buf_pa;
```

**Step 12 — Write TX ring base (high before low)**
```c
mmio_w32(mmio, RTL_TXDESC_ADDR_HIGH, (uint32_t)(tx_ring_phys >> 32));
mmio_w32(mmio, RTL_TXDESC_ADDR_LOW,  (uint32_t)(tx_ring_phys));
```
High word must be written first — Linux comment: "some ARM board needs TxDescAddrHigh written before TxDescAddrLow to work."

**Step 13 — Write RX ring base (high before low)**
```c
mmio_w32(mmio, RTL_RXDESC_ADDR_HIGH, (uint32_t)(rx_ring_phys >> 32));
mmio_w32(mmio, RTL_RXDESC_ADDR_LOW,  (uint32_t)(rx_ring_phys));
```

**Step 14 — MaxTxPacketSize (early-TX threshold)**
Write `0x3Fu` to `RTL_MAXTXPKTSIZE`. Value `0x3F` = `NoEarlyTx` — disables early TX. This is a threshold register, not a hard cap on frame size.

**Step 15 — Cfg9346 lock**
Write `RTL_CFG9346_LOCK` (0x00) to `RTL_CFG9346`. Re-lock config registers.

**Step 16 — Enable RX/TX**
Write `RTL_CMD_RXEN | RTL_CMD_TXEN` (0x0C) to `RTL_CHIPCMD`.

**Step 17 — RxConfig**
Write:
```c
RTL_RX128_INT_EN | RTL_RX_MULTI_EN | RTL_RX_DMA_BURST |
RTL_RX_EARLY_OFF | RTL_ACCEPT_BCAST | RTL_ACCEPT_MYPHYS
```

**Step 18 — TxConfig**
Write `RTL_TXCONFIG_VAL` to `RTL_TXCONFIG`.

**Step 19 — Register netdev**
Fill `netdev_t`: name `eth0`/`eth1`, mtu=1500, `send=rtl8168_send`, `poll=rtl8168_poll`, `priv=&s_devs[s_count]`. Call `netdev_register()`.

**Step 20 — Announce**
```c
printk("[NET] OK: rtl8168 eth%d mac=%02x:%02x:%02x:%02x:%02x:%02x\n", ...);
```

### TX path (`rtl8168_send`)

```
slot = tx_head & 255
copy frame into tx_buf[slot]   /* raw Ethernet frame, no header prepend */
tx_ring[slot].opts2 = 0
tx_ring[slot].addr  = tx_buf_phys[slot]   /* cached at init — no page-table walk */
tx_ring[slot].opts1 = RTL_DESC_OWN | RTL_DESC_FIRST_FRAG | RTL_DESC_LAST_FRAG | len
                      | (slot == 255 ? RTL_DESC_RING_END : 0)
compiler barrier              /* prevent reorder of opts1 write past TxPoll write */
mmio_w8(mmio, RTL_TXPOLL, RTL_NPQ)   /* kick NIC to scan TX ring */
poll: wait for (tx_ring[slot].opts1 & RTL_DESC_OWN) == 0 (max 100,000 iters)
tx_head++
return 0 on success, -1 on timeout
```

### RX path (`rtl8168_poll`, called from PIT at 100 Hz)

```
while (rx_ring[rx_head & 255].opts1 & RTL_DESC_OWN) == 0:
    compiler barrier
    opts1 = rx_ring[rx_head & 255].opts1
    len   = opts1 & RTL_DESC_LEN_MASK   /* bits[13:0], mask = 0x3FFF */
    slot  = rx_head & 255
    if len > 14:
        netdev_rx_deliver(dev, rx_buf[slot], (uint16_t)len)
    /* re-arm: give descriptor back to NIC */
    rx_ring[slot].opts2 = 0
    rx_ring[slot].opts1 = RTL_DESC_OWN | RTL_RX_BUF_SIZE
                          | (slot == 255 ? RTL_DESC_RING_END : 0)
    rx_head++
```

## PCI Device Scan

`rtl8168_init()` must iterate the PCI device table using the `pcie_get_devices()` + loop pattern with explicit `vendor_id == 0x10ECu && device_id == 0x8168u` check — same pattern as `virtio_net_init()`. Do **not** use `pcie_find_device()` (matches by class only; would also match virtio-net and any other Ethernet controller).

```c
int count = pcie_device_count();
for (i = 0; i < count && s_count < RTL8168_MAX_DEVS; i++) {
    const pcie_device_t *d = &pcie_get_devices()[i];
    if (d->vendor_id == 0x10ECu && d->device_id == 0x8168u)
        rtl8168_init_one(d);
}
```

## Naming Constraint

`netdev_register()` assigns names sequentially: first call → `eth0`, second → `eth1`. `virtio_net_init()` always runs first in `main.c` and would claim `eth0` if a virtio-net device is present. On the target bare-metal hardware (no virtio-net), `virtio_net_init()` returns silently and `rtl8168_init()` correctly claims `eth0`/`eth1`. On a q35 QEMU machine with virtio-net + emulated RTL8168, both drivers would try to register `eth0` — second registration would get `eth1`. This is acceptable for now and noted as a forward constraint.

## Testing

No automated test (RTL8168 not present in QEMU). `make test` must pass unchanged — driver is fully silent if `10ec:8168` not found.

Manual test on target hardware:
1. Write ISO to USB, boot target machine
2. Capture COM1 serial output
3. Verify: `[NET] OK: rtl8168 eth0 mac=...` and `[NET] OK: rtl8168 eth1 mac=...`
4. Verify Phase 25 ICMP: host ping to `10.0.2.15` succeeds (ARP reply + ICMP echo reply observed on host)
