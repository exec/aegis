# Phase 24: AMD DC Framebuffer Backend (Ryzen 5000-7000) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement an AMD Display Core (DC) framebuffer backend that drives the integrated GPU on Ryzen 5000-7000 series processors, enabling native-resolution display output on real hardware without relying on GRUB's VESA/GOP framebuffer.

**Architecture:** The AMD DC backend (`kernel/arch/x86_64/fb_amddc.c`) probes for an AMD GPU via PCIe (vendor 0x1002, class 0x03), reads the DCN IP version register to determine the display controller generation (DCN 2.1, 3.0, or 3.1), reads the monitor's EDID via DDC I2C (`i2c_ddc.c`) to discover native resolution and timing, allocates a scanout framebuffer in VRAM (BAR5 aperture), and programs the display pipeline (HUBP, DPP, OPP, OPTC, DIO) to output pixels. The backend registers an `fb_info_t` with the framebuffer abstraction layer. On QEMU, the AMD DC probe returns 0 and the VESA backend is used instead.

**Tech Stack:** C, AMD DCN 2.1/3.0/3.1 register tables, PCIe BAR0 (register MMIO) + BAR5 (VRAM aperture), DDC I2C master registers, EDID parsing, x86-64 MMIO.

- [ ] **Step 1: Create i2c_ddc.h**

```c
/* i2c_ddc.h — DDC I2C master for EDID reading on AMD DC
 *
 * Uses the I2C master built into AMD display controllers.
 * Register offsets differ per DCN version; a dispatch table
 * selects the correct offsets based on the DCN major.minor.
 */
#ifndef I2C_DDC_H
#define I2C_DDC_H

#include <stdint.h>

/* EDID block — 128 bytes */
typedef struct __attribute__((packed)) {
    uint8_t  header[8];         /* 00 FF FF FF FF FF FF 00 */
    uint16_t manufacturer;      /* big-endian compressed ASCII */
    uint16_t product_code;
    uint32_t serial;
    uint8_t  mfg_week;
    uint8_t  mfg_year;          /* year - 1990 */
    uint8_t  edid_version;
    uint8_t  edid_revision;
    uint8_t  input_params;
    uint8_t  width_cm;
    uint8_t  height_cm;
    uint8_t  gamma;
    uint8_t  features;
    uint8_t  chroma[10];
    uint8_t  established_timings[3];
    uint8_t  standard_timings[16];
    uint8_t  detailed_timings[72]; /* 4 x 18-byte detailed timing descriptors */
    uint8_t  num_extensions;
    uint8_t  checksum;
} edid_block_t;

/* Parsed display timing from EDID detailed timing descriptor */
typedef struct {
    uint32_t pixel_clock_khz;
    uint16_t h_active;
    uint16_t h_blank;
    uint16_t h_sync_offset;
    uint16_t h_sync_width;
    uint16_t v_active;
    uint16_t v_blank;
    uint16_t v_sync_offset;
    uint16_t v_sync_width;
} edid_timing_t;

/* Read 128-byte EDID block via DDC I2C.
 * bar0: mapped BAR0 base of AMD GPU
 * dcn_major, dcn_minor: DCN version for register offset selection
 * edid_out: pointer to 128-byte buffer
 * Returns 0 on success, -1 on failure. */
int i2c_ddc_read_edid(volatile uint8_t *bar0,
                       uint32_t dcn_major, uint32_t dcn_minor,
                       edid_block_t *edid_out);

/* Parse the first detailed timing descriptor from an EDID block.
 * Returns 0 on success, -1 if no valid timing found. */
int edid_parse_timing(const edid_block_t *edid, edid_timing_t *timing_out);

#endif /* I2C_DDC_H */
```

- [ ] **Step 2: Create i2c_ddc.c**

Implement the AMD DC I2C master register access. The I2C master registers differ per DCN version — use a register offset table:

```c
/* i2c_ddc.c — AMD DC I2C master for EDID reading */
#include "i2c_ddc.h"
#include "printk.h"
#include <string.h>

/* I2C register offsets per DCN version */
typedef struct {
    uint32_t setup;         /* DC_I2C_DDC_SETUP */
    uint32_t speed;         /* DC_I2C_DDC_SPEED */
    uint32_t control;       /* DC_I2C_CONTROL */
    uint32_t status;        /* DC_I2C_STATUS */
    uint32_t data;          /* DC_I2C_DATA */
} i2c_regs_t;

/* DCN 2.1 (Cezanne) register offsets — sourced from references/amd-dc/ */
static const i2c_regs_t dcn21_i2c = {
    .setup   = 0x1E4C,
    .speed   = 0x1E50,
    .control = 0x1E54,
    .status  = 0x1E58,
    .data    = 0x1E5C,
};

/* DCN 3.0 (Rembrandt) */
static const i2c_regs_t dcn30_i2c = {
    .setup   = 0x1E4C,     /* verify from reference */
    .speed   = 0x1E50,
    .control = 0x1E54,
    .status  = 0x1E58,
    .data    = 0x1E5C,
};

/* DCN 3.1 (Phoenix) */
static const i2c_regs_t dcn31_i2c = {
    .setup   = 0x1E4C,     /* verify from reference */
    .speed   = 0x1E50,
    .control = 0x1E54,
    .status  = 0x1E58,
    .data    = 0x1E5C,
};
```

The I2C transaction sequence:
1. Write slave address (0xA0 for EDID, 0xA1 for read)
2. Write byte offset (0x00 for first 128 bytes)
3. Trigger read of 128 bytes
4. Poll status register for completion
5. Read 128 bytes from data register
6. Verify EDID header: bytes 0-7 must be `00 FF FF FF FF FF FF 00`
7. Verify checksum: sum of all 128 bytes must be 0

Implement `edid_parse_timing()`:
1. Find first detailed timing descriptor in `edid->detailed_timings[0..17]`
2. Parse pixel clock (bytes 0-1, little-endian, units of 10 kHz)
3. Parse horizontal active, blank, sync offset, sync width
4. Parse vertical active, blank, sync offset, sync width
5. Fill `edid_timing_t`

- [ ] **Step 2: Create fb_amddc.c — part 1: PCIe probe and DCN version detect**

```c
/* fb_amddc.c — AMD Display Core framebuffer backend */
#include "fb_amddc.h"
#include "i2c_ddc.h"
#include "pcie.h"
#include "printk.h"
#include "vmm.h"
#include "kva.h"
#include "pmm.h"
#include <string.h>

/* IP_VERSION register offset from BAR0 */
#define AMD_IP_VERSION_REG   0x2600C

/* DCN register offset tables per version */
typedef struct {
    /* HUBP (Hub and Prefetch) */
    uint32_t hubp_surface_addr_lo;
    uint32_t hubp_surface_addr_hi;
    uint32_t hubp_surface_pitch;
    uint32_t hubp_surface_config;
    /* DPP (Display Pipe and Plane) */
    uint32_t dpp_control;
    /* OPP (Output Pixel Processing) */
    uint32_t opp_control;
    /* OPTC (Output Pipe and Timing Controller) */
    uint32_t optc_h_total;
    uint32_t optc_h_blank;
    uint32_t optc_h_sync;
    uint32_t optc_v_total;
    uint32_t optc_v_blank;
    uint32_t optc_v_sync;
    uint32_t optc_control;
    /* DIO (Display I/O) */
    uint32_t dio_encoder_control;
    uint32_t dio_dp_link_control;
} dcn_regs_t;
```

Implement `fb_amddc_probe()`:

1. Scan PCIe devices for vendor 0x1002, class 0x03:
   ```c
   const pcie_device_t *gpu = NULL;
   int i;
   for (i = 0; i < pcie_device_count(); i++) {
       const pcie_device_t *d = &pcie_get_devices()[i];
       if (d->vendor_id == 0x1002 && d->class_code == 0x03) {
           gpu = d; break;
       }
   }
   if (!gpu) return 0;
   ```

2. Map BAR0 (register space, typically 16-32MB) into kernel VA

3. Read IP_VERSION register at BAR0 + 0x2600C:
   ```c
   uint32_t ip_ver = *(volatile uint32_t *)(bar0_va + AMD_IP_VERSION_REG);
   uint32_t major = (ip_ver >> 16) & 0xFF;
   uint32_t minor = (ip_ver >> 8) & 0xFF;
   ```

4. Validate supported DCN versions: 2.1, 3.0, 3.1. If unsupported:
   ```c
   printk("[FB] WARN: AMD DC unsupported DCN %u.%u, falling back to VESA\n",
          major, minor);
   return 0;
   ```

5. Select DCN register offset table based on version

6. Map BAR5 (VRAM aperture) into kernel VA

- [ ] **Step 1: Add EDID read and timing parse**

```c
    if (i2c_ddc_read_edid(bar0_va, major, minor, &edid) == 0 &&

        /* Use native timing from EDID */
```

- [ ] **Step 2: Allocate scanout framebuffer in VRAM**

Allocate from BAR5 base, aligned to 256 bytes:

```c
    uint32_t fb_size = timing.h_active * timing.v_active * 4;  /* ARGB8888 */
    uint64_t fb_vram_offset = 0;  /* start of VRAM aperture */
    /* Ensure 256-byte alignment */
    fb_vram_offset = (fb_vram_offset + 255) & ~255ULL;
```

Map the VRAM framebuffer region into kernel VA for CPU access.

- [ ] **Step 3: Program HUBP (Hub and Prefetch)**

Configure the HUBP to fetch pixels from the VRAM scanout buffer:

```c
    /* Set surface address (VRAM physical, relative to BAR5 base) */
    mmio_write32(bar0_va + regs->hubp_surface_addr_lo,
                 (uint32_t)(fb_vram_offset & 0xFFFFFFFF));
    mmio_write32(bar0_va + regs->hubp_surface_addr_hi,
                 (uint32_t)(fb_vram_offset >> 32));
    /* Set pitch in pixels */
    mmio_write32(bar0_va + regs->hubp_surface_pitch, timing.h_active);
    /* Set pixel format: ARGB8888 */
    mmio_write32(bar0_va + regs->hubp_surface_config, 0x2);  /* format code */
```

- [ ] **Step 4: Program DPP (Display Pipe and Plane)**

Set identity color matrix, no scaling:

```c
    mmio_write32(bar0_va + regs->dpp_control, 0x0);  /* bypass mode */
```

- [ ] **Step 5: Program OPP (Output Pixel Processing)**

Bypass LUT:

```c
    mmio_write32(bar0_va + regs->opp_control, 0x0);  /* bypass */
```

- [ ] **Step 6: Program OPTC (Output Pipe and Timing Controller)**

Program CRTC timings from EDID:

```c
    uint32_t h_total = timing.h_active + timing.h_blank;
    uint32_t v_total = timing.v_active + timing.v_blank;

    mmio_write32(bar0_va + regs->optc_h_total, h_total - 1);
    mmio_write32(bar0_va + regs->optc_h_blank,
                 ((timing.h_active + timing.h_blank - 1) << 16) |
                  timing.h_active);
    mmio_write32(bar0_va + regs->optc_h_sync,
                 ((timing.h_sync_offset + timing.h_sync_width - 1) << 16) |
                  timing.h_sync_offset);
    mmio_write32(bar0_va + regs->optc_v_total, v_total - 1);
    mmio_write32(bar0_va + regs->optc_v_blank,
                 ((timing.v_active + timing.v_blank - 1) << 16) |
                  timing.v_active);
    mmio_write32(bar0_va + regs->optc_v_sync,
                 ((timing.v_sync_offset + timing.v_sync_width - 1) << 16) |
                  timing.v_sync_offset);
    mmio_write32(bar0_va + regs->optc_control, 0x1);  /* enable */
```

- [ ] **Step 7: Program DIO (Display I/O)**

Enable the encoder for DisplayPort or HDMI:

```c
#include "fb_amddc.h"

void fb_init(void)
{
    /* Try AMD DC first (real hardware) */
    if (fb_amddc_probe()) {
        /* s_active_fb set by amddc backend */
        s_active_fb = fb_amddc_get_info();
        s_con_cols = s_active_fb->width / FB_FONT_WIDTH;
        s_con_rows = s_active_fb->height / FB_FONT_HEIGHT;
        return;  /* [FB] OK: already printed by amddc */
    }

    /* Fall back to VESA/GOP */
    if (fb_vesa_init()) {
        s_active_fb = fb_vesa_get_info();
        s_con_cols = s_active_fb->width / FB_FONT_WIDTH;
        s_con_rows = s_active_fb->height / FB_FONT_HEIGHT;
        printk("[FB] OK: %ux%u bpp=%u\n",
               s_active_fb->width, s_active_fb->height, s_active_fb->bpp);
        return;
    }

    printk("[FB] OK: no framebuffer tag (text-mode only)\n");
}
```

- [ ] **Step 2: Add Phase 24 forward-looking constraints**

```markdown
### Phase 24 forward-looking constraints

**Real-hardware only testing.** AMD DC cannot be tested in QEMU — there is no AMD GPU emulation. The probe returns 0 on QEMU and the VESA backend is used. All AMD DC testing requires a physical Ryzen 5000-7000 machine.

**DCN register offsets are approximate.** The register offset tables are sourced from Linux amd-dc reference files but may need adjustment per silicon stepping. If the display does not light up on a specific Ryzen model, dump BAR0 registers and compare with reference values.

**No clock configuration.** The AMD DC init assumes display clocks (DCLK, DISPCLK) are already running at firmware-set frequencies. No PLL programming. If firmware leaves clocks off, the display will not work.

**No power management.** Display power states (DPMS, panel power sequencing for eDP) are not managed. The display stays on at full brightness.

**Single output only.** The first detected connected output (DisplayPort or HDMI) is used. Multi-display is v2.0 work.

**No HDMI audio.** Only video output is initialized. HDMI audio requires separate audio codec programming.

**VRAM allocation is a fixed offset.** The scanout buffer is placed at offset 0 in the VRAM aperture. No VRAM allocator — additional VRAM consumers (cursor, overlay) must use manually chosen offsets.
```
