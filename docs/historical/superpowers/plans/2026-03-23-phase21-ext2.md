# Phase 21: ext2 Read-Write Filesystem Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a read-write ext2 filesystem driver that mounts the NVMe block device as the root filesystem, enabling persistent file storage across reboots.

**Architecture:** The ext2 driver (`kernel/fs/ext2.c`) obtains a `blkdev_t` pointer via `blkdev_get("nvme0")` and reads the superblock at byte offset 1024. A 16-slot LRU block cache sits between ext2 and the block device, caching reads and writing back dirty blocks on eviction. Path resolution walks directory entries from root inode 2. The VFS layer gains ext2-backed `open`, `read`, `write`, `create`, and `unlink` operations. New shell commands (`mkdir`, `touch`, `rm`, `cp`, `mv`) exercise the write path.

**Tech Stack:** C, ext2 on-disk structures (superblock, block group descriptors, inodes, directory entries), blkdev read/write, KVA allocator for block cache, `e2tools` for disk population.

Define all on-disk structures needed for ext2:

```c
/* ext2.h — ext2 filesystem driver
 *
 * Read-write ext2 over blkdev. No journal (ext2, not ext3/4).
 * Block cache: 16-slot LRU. Single indirect blocks supported.
 */
#ifndef EXT2_H
#define EXT2_H

#include <stdint.h>

#define EXT2_MAGIC          0xEF53
#define EXT2_ROOT_INODE     2
#define EXT2_DIRECT_BLOCKS  12
#define EXT2_NAME_LEN       255

/* Superblock — at byte offset 1024 from partition start */
typedef struct __attribute__((packed)) {
    uint32_t s_inodes_count;
    uint32_t s_blocks_count;
    uint32_t s_r_blocks_count;
    uint32_t s_free_blocks_count;
    uint32_t s_free_inodes_count;
    uint32_t s_first_data_block;
    uint32_t s_log_block_size;      /* block_size = 1024 << s_log_block_size */
    uint32_t s_log_frag_size;
    uint32_t s_blocks_per_group;
    uint32_t s_frags_per_group;
    uint32_t s_inodes_per_group;
    uint32_t s_mtime;
    uint32_t s_wtime;
    uint16_t s_mnt_count;
    uint16_t s_max_mnt_count;
    uint16_t s_magic;
    uint16_t s_state;
    uint16_t s_errors;
    uint16_t s_minor_rev_level;
    uint32_t s_lastcheck;
    uint32_t s_checkinterval;
    uint32_t s_creator_os;
    uint32_t s_rev_level;
    uint16_t s_def_resuid;
    uint16_t s_def_resgid;
    /* EXT2_DYNAMIC_REV fields */
    uint32_t s_first_ino;
    uint16_t s_inode_size;
    uint16_t s_block_group_nr;
    uint32_t s_feature_compat;
    uint32_t s_feature_incompat;
    uint32_t s_feature_ro_compat;
    uint8_t  s_uuid[16];
    char     s_volume_name[16];
    char     s_last_mounted[64];
    uint32_t s_algo_bitmap;
    /* Padding to 1024 bytes total — we only read what we need */
} ext2_superblock_t;

/* Block Group Descriptor — 32 bytes */
typedef struct __attribute__((packed)) {
    uint32_t bg_block_bitmap;
    uint32_t bg_inode_bitmap;
    uint32_t bg_inode_table;
    uint16_t bg_free_blocks_count;
    uint16_t bg_free_inodes_count;
    uint16_t bg_used_dirs_count;
    uint16_t bg_pad;
    uint8_t  bg_reserved[12];
} ext2_bgd_t;

/* Inode — 128 bytes (ext2 rev 0) */
typedef struct __attribute__((packed)) {
    uint16_t i_mode;
    uint16_t i_uid;
    uint32_t i_size;
    uint32_t i_atime;
    uint32_t i_ctime;
    uint32_t i_mtime;
    uint32_t i_dtime;
    uint16_t i_gid;
    uint16_t i_links_count;
    uint32_t i_blocks;          /* 512-byte sectors */
    uint32_t i_flags;
    uint32_t i_osd1;
    uint32_t i_block[15];       /* [0..11] direct, [12] indirect, [13] double, [14] triple */
    uint32_t i_generation;
    uint32_t i_file_acl;
    uint32_t i_dir_acl;
    uint32_t i_faddr;
    uint8_t  i_osd2[12];
} ext2_inode_t;

/* Directory entry */
typedef struct __attribute__((packed)) {
    uint32_t inode;
    uint16_t rec_len;
    uint8_t  name_len;
    uint8_t  file_type;
    char     name[EXT2_NAME_LEN];
} ext2_dirent_t;

/* File type values in directory entries */
#define EXT2_FT_UNKNOWN  0
#define EXT2_FT_REG_FILE 1
#define EXT2_FT_DIR      2

/* Inode mode bits */
#define EXT2_S_IFMT   0xF000
#define EXT2_S_IFREG  0x8000
#define EXT2_S_IFDIR  0x4000

/* Mount an ext2 filesystem from the named block device.
 * Returns 0 on success, -1 on failure. */
int ext2_mount(const char *devname);

/* File operations for VFS integration */
int ext2_open(const char *path, uint32_t *inode_out);
int ext2_read(uint32_t inode_num, void *buf, uint32_t offset, uint32_t len);
int ext2_write(uint32_t inode_num, const void *buf, uint32_t offset, uint32_t len);
int ext2_create(const char *path, uint16_t mode);
int ext2_unlink(const char *path);
int ext2_mkdir(const char *path, uint16_t mode);
int ext2_file_size(uint32_t inode_num);
int ext2_readdir(uint32_t dir_inode, void *buf, uint32_t buf_size);

#endif /* EXT2_H */
```

### Task 2: Implement ext2.c — mount, block cache, read path

**Files:**
- Create: `kernel/fs/ext2.c`

- [ ] **Step 1: Implement block cache**

16-slot LRU cache. Each slot: `{ uint32_t block_num; uint8_t data[block_size]; uint8_t dirty; uint32_t age; }`. On cache miss: find LRU slot (lowest age), write back if dirty, read new block via `blkdev_read`. On cache hit: increment age counter.

- [ ] **Step 2: Implement ext2_mount**

1. Call `blkdev_get(devname)` to get the `blkdev_t`
2. Read superblock (byte offset 1024 = LBA 2 for 512-byte sectors)
3. Verify magic 0xEF53
4. Compute `block_size = 1024 << sb.s_log_block_size`
5. Compute block group count: `(sb.s_blocks_count + sb.s_blocks_per_group - 1) / sb.s_blocks_per_group`
6. Read block group descriptor table (block 2 for 1024-byte blocks, block 1 for 4096-byte blocks)
7. Store superblock and BGD table in static globals
8. Print `[EXT2] OK: mounted %s, %u blocks, %u inodes`

- [ ] **Step 3: Implement inode read**

`ext2_read_inode(uint32_t ino, ext2_inode_t *out)`:
1. Compute block group: `(ino - 1) / sb.s_inodes_per_group`
2. Compute index within group: `(ino - 1) % sb.s_inodes_per_group`
3. Look up `bg_inode_table` from BGD
4. Compute byte offset within inode table: `index * sb.s_inode_size`
5. Read via block cache

- [ ] **Step 4: Implement block number resolution**

`ext2_block_num(ext2_inode_t *inode, uint32_t file_block)`:
- If `file_block < 12`: return `inode->i_block[file_block]` (direct)
- If `file_block < 12 + (block_size/4)`: read indirect block `inode->i_block[12]`, index by `file_block - 12`
- Double/triple indirect: return -1 (unsupported in v1.0)

- [ ] **Step 5: Implement ext2_open (path walk)**

Walk from root inode (inode 2). For each path component, scan directory entries in the current directory inode. Match by name. Return the final inode number.

- [ ] **Step 6: Implement ext2_read**

Read data blocks via block cache, copying into the caller's buffer. Handle partial first/last block reads. Use `ext2_block_num()` to resolve file block indices to disk block numbers.

### Task 3: Implement ext2 write path

**Files:**
- Modify: `kernel/fs/ext2.c`

- [ ] **Step 1: Implement block allocation**

`ext2_alloc_block(uint32_t preferred_group)`:
1. Scan block bitmap of preferred group for a free bit
2. If none, try subsequent groups
3. Set the bit, decrement `bg_free_blocks_count` and `sb.s_free_blocks_count`
4. Write back bitmap block (mark dirty in cache)
5. Return allocated block number

- [ ] **Step 2: Implement inode allocation**

`ext2_alloc_inode(uint32_t preferred_group)`:
1. Scan inode bitmap of preferred group for a free bit
2. Set the bit, decrement `bg_free_inodes_count` and `sb.s_free_inodes_count`
3. Write back bitmap block
4. Return allocated inode number

- [ ] **Step 3: Implement ext2_write**

Write data to an inode's data blocks:
1. For each file block in the write range, check if `i_block[n]` is 0 (unallocated)
2. If unallocated, call `ext2_alloc_block()` and set `i_block[n]`
3. Read block via cache, modify bytes, mark dirty
4. Update `i_size` if write extends the file
5. Update `i_blocks` (512-byte sector count)
6. Write back inode

- [ ] **Step 4: Implement ext2_create**

Create a new regular file:
1. Walk parent directory path
2. Allocate new inode via `ext2_alloc_inode()`
3. Initialize inode: mode, size=0, links_count=1, zero i_block[]
4. Write inode to disk
5. Add directory entry to parent: find space in last directory block or allocate new block
6. Return new inode number

- [ ] **Step 5: Implement ext2_unlink**

Remove a file:
1. Walk parent directory path, find the dirent for the target name
2. Read target inode, decrement `i_links_count`
3. If links_count == 0: free all data blocks (clear bitmap bits), free inode (clear bitmap bit)
4. Remove directory entry: set `inode` field to 0, merge rec_len with previous entry

- [ ] **Step 6: Implement ext2_mkdir**

Create a directory:
1. Allocate inode, set mode with `EXT2_S_IFDIR`
2. Allocate one data block for `.` and `..` entries
3. Write `.` entry (self inode) and `..` entry (parent inode)
4. Add entry in parent directory
5. Increment parent `i_links_count` (for `..`)
6. Increment `bg_used_dirs_count`

- [ ] **Step 7: Implement cache flush**

`ext2_sync()`: walk all 16 cache slots, write back any dirty blocks. Called before shutdown and periodically.

```c
static int64_t sys_mkdir(const char *path, int mode)
{
    char kpath[256];
    if (copy_from_user(kpath, path, sizeof(kpath)) < 0)
        return -14; /* EFAULT */
    kpath[255] = '\0';
    (void)mode;
    return ext2_mkdir(kpath);
}

static int64_t sys_unlink(const char *path)
{
    char kpath[256];
    if (copy_from_user(kpath, path, sizeof(kpath)) < 0)
        return -14; /* EFAULT */
    kpath[255] = '\0';
    return ext2_unlink(kpath);
}

static int64_t sys_rename(const char *old_path, const char *new_path)
{
    char kold[256], knew[256];
    if (copy_from_user(kold, old_path, sizeof(kold)) < 0) return -14;
    if (copy_from_user(knew, new_path, sizeof(knew)) < 0) return -14;
    kold[255] = '\0'; knew[255] = '\0';
    return ext2_rename(kold, knew);
}
```

Also add `ext2_mkdir()`, `ext2_unlink()`, `ext2_rename()` declarations to `kernel/fs/ext2.h`.

```markdown
### Phase 21 forward-looking constraints

**No double/triple indirect blocks.** Files larger than ~268KB (12 direct + 256 indirect blocks at 1024 bytes each) cannot be accessed. Adequate for shell scripts and small programs; large files require double-indirect support.

**Block cache is 16 slots.** Heavy random I/O will thrash the cache. A page cache is v2.0 work.

**No fsck on mount.** Dirty unmount may leave inconsistent state. `fsck.ext2` from host recovers.

**ext2_mount uses raw device "nvme0".** No partition table parsing. Phase 25 switches to "nvme0p2" after GPT parsing.

**No timestamps.** inode atime/mtime/ctime are always 0. A wall clock source is needed for real timestamps.

**New build dependency: `e2tools`.** Used by `make disk` to populate the ext2 image.
```
